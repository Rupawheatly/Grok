### Experiment 2 — Network Depth (H1, Section 5.2)

**Paper output:** Table 2, Figure 2

**Notebook:** `Exp_H1_Depth8_Complete.ipynb`

Completes the 5-seed depth-8 residual MLP experiment by running seeds 3 and 4,
which were missing from the initial 3-seed run. Seeds 0–2 results are hardcoded
from prior runs; seeds 3 and 4 are trained fresh.

| What it produces | File |
|---|---|
| Per-seed delays for depth-8 (5 seeds) | printed to stdout |
| Updated Table 2 stats (mean=33,333 ± 12,858) | printed to stdout |
| Figure: depth-8 bar chart | `fig_h1_cell6.png` |

**Key result:** Seeds 3 and 4 both DNF → depth-8 grokks in 3/5 seeds.
The non-monotonic depth relationship (depth-4 = 0/5, depth-8 = 3/5) is confirmed.

**Notebook:** `Exp_H2_Corrected_Configs.ipynb`

Reruns the MLP vs Transformer comparison at the exact hyperparameter configurations

produced a misleading 2.18× gap.

| What it produces | File |

### Experiment 4 — Activation Function (H3, Section 5.4)

**Paper output:** Table 4 (Sweep A and B), Figure 4

**Notebook:** `Exp_H3_Fixed_Baselines.ipynb`

Tests GELU, ReLU, and Tanh at two hyperparameter configurations:
- **Sweep A** (lr=1e-2, λ=2e-3, width=256): the originally planned H3 config
- **Sweep B** (lr=3e-2, λ=5e-4, width=512): the H2 MLP baseline config

5 seeds per activation per sweep = 30 total runs.

| What it produces | File |
|---|---|
| Per-seed delays for all 6 conditions | printed to stdout |
| Sweep A: GELU 0/5, ReLU 2/5, Tanh 3/5 | `sweep_A` rows in stdout |
| Sweep B: GELU 5/5 (45,600 steps), ReLU 5/5 (196,800), Tanh 2/5 | `sweep_B` rows |
| GELU/ReLU ratio: 4.32× in Sweep B | printed verdict |
| Figure: 2-panel sweep comparison | `fig_h3_cell6.png` |

**Key result:** GELU's advantage is config-dependent. At Sweep A, GELU fails
entirely (0/5). At Sweep B it wins by 4.32×. The activation–regularisation
interaction is the paper's most novel empirical finding.

**Hardware:** T4 GPU, CUDA graphs.

### Experiment 5 — Weight Decay Sweep (H4, Section 5.5)

**Paper output:** Tables 5a and 5b, Figure 5

**Notebook:** `Exp_H4_WD_Sweep.ipynb`

Three-part experiment:
- **Part A:** MLP λ sweep over {1e-5, 5e-5, 1e-4, 5e-4, 1e-3, 2e-3, 5e-3},
  3 screening seeds, SGD lr=3e-2
- **Part B:** Transformer λ sweep over {0.01, 0.1, 0.5, 1.0, 2.0, 5.0},
  3 screening seeds, AdamW lr=1e-3
- **Part C:** Fair comparison at optimal λ each (MLP λ=1e-3, TF λ=5.0),
  5 confirmation seeds each

| What it produces | File |
|---|---|
| MLP sweep summary (optimal λ=1e-3, mean=25,333 ± 5,033) | stdout |
| Transformer sweep summary (optimal λ=5.0, mean=24,000 ± 10,583) | stdout |
| Part C definitive ratio: 1.90× at optimal λ each | stdout |
| Figure: 3-panel sweep + fair comparison | `fig_h4_nb_cell8.png` |

**Key result:** Optimal λ differs by 5,000× between architectures (1e-3 vs 5.0).
λ=2e-3 collapses MLP training entirely — a sharp Goldilocks cliff.
The definitive architecture gap is **1.90×**, correcting H2's 1.11×.

**Hardware:** T4 GPU, CUDA graphs.

### Experiment 6 — Weight Norm Dynamics (Section 5.6)

**Paper output:** Table 6, Figure 6

**Notebook:** `Exp_WeightNorm_Dynamics.ipynb`

Trains 6 key configurations (1 seed each) and logs RMS parameter norm,
train accuracy, and test accuracy every 2,000 steps. Weight norm is computed
inside the CUDA graph — zero CPU overhead.

Configurations:
1. Depth-2 GELU baseline (lr=1e-2, λ=2e-3, width=256)
2. MLP GELU at H2 config (lr=3e-2, λ=5e-4, width=512)
3. MLP GELU at optimal λ (lr=3e-2, λ=1e-3, width=512)
4. Transformer at H2 λ=1.0 (AdamW lr=1e-3)
5. Transformer at optimal λ=5.0 (AdamW lr=1e-3)
6. MLP ReLU at Sweep-B config (lr=3e-2, λ=5e-4, width=512)

| What it produces | File |
|---|---|
| Weight norm at grokking step for each config | stdout table |
| Mean ± std across width-512 configs (0.022 ± 0.003, CV=14.5%) | stdout |
| Figure: 6-panel (accuracy, weight norm, threshold bar, 3 training curves) | `fig_weightnorm_cell3.png` |

**Key result:** All five width-512 models grokk at the same weight norm threshold
(~0.022, CV=14.5%) regardless of architecture, activation, or optimizer.
GELU and ReLU reach the threshold at identical values but GELU arrives 6.9×
faster — activation controls *rate*, not *threshold*.

**Hardware:** T4 GPU, CUDA graphs.

### Experiment 7 — Fourier Analysis (Section 5.7)

**Paper output:** Table 7, Figure 7

**Notebook:** `Exp_Fourier_Analysis.ipynb`

Trains three models to grokking (MLP-GELU, MLP-ReLU, Transformer at optimal
λ each, seed=0), then runs the Fourier analysis on their learned embedding
matrices. No new training needed after the three seeds complete.

| What it produces | File |
|---|---|
| Top-5 Fourier concentration per model | stdout |
| MLP-GELU: 74.7%, MLP-ReLU: 75.6%, Transformer: 98.5% | stdout |
| Top-5 dominant frequencies per model | stdout |
| Figure: 6-panel (3 spectra + 3 embedding projections) | `fig_fourier_cell4.png` |

**Key result:** All three models converge to sparse Fourier representations
post-grokking (confirms Nanda et al. 2023). Transformer is far more concentrated
(98.5% vs ~75%). GELU and ReLU converge to equally sparse representations
despite a 3× timing gap — activation does not change *what* solution is learned.

**Hardware:** T4 GPU. Runtime ~5 min training + instant analysis.

### Experiment 8 — Modular Multiplication (Section 5.8)

**Paper output:** Tables 8a and 8b, Figure 8

**Notebook:** `Exp_Mod_Multiplication.ipynb`

Replaces `(a+b) mod 97` with `(a×b) mod 97` and repeats H2 and H3 at
optimal λ (MLP λ=1e-3, Transformer λ=5.0). 5 seeds each condition.

| What it produces | File |
|---|---|
| H2-mul: MLP mean=24,800 ± 3,347 vs TF mean=35,200 ± 17,470 | stdout |
| Ratio on multiplication: 1.42× (vs 1.90× on addition) | stdout |
| H3-mul: GELU 24,400, ReLU 93,200, Tanh 3/5 (102,667) | stdout |
| GELU/ReLU ratio: 3.82× (vs 4.32× on addition) | stdout |
| Figure: 3-panel (architecture, activation, addition vs multiplication) | `fig_modmul_cell3.png` |

**Key result:** Both the architecture gap and GELU advantage replicate on
multiplication, confirming task-independence. Both architectures are *faster*
on multiplication than addition at optimal λ — Transformer benefits more
(0.69× of its addition time vs 0.91× for MLP).

**Hardware:** T4 GPU, CUDA graphs.

## Hardware and reproducibility

All experiments use:
- **GPU:** NVIDIA T4 (Google Colab / Kaggle)
- **Framework:** PyTorch 2.x with `torch.cuda.CUDAGraph` acceleration
- **Seeds:** Fixed per run (see each notebook's `torch.manual_seed()` call)
- **Data:** Modular arithmetic mod 97 — generated deterministically from seeds,
  no external download required

To reproduce any experiment exactly: upload the corresponding notebook to
Colab or Kaggle, select T4 GPU, and run all cells. Each notebook is fully
self-contained — no shared state between notebooks.
